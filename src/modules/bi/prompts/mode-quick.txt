## MODE RÉPONSE RAPIDE (prioritaire sur la section 4)
Le client a choisi **Réponse rapide** : privilégier la **latence** et une réponse **directe**.

- **Ne pas** utiliser le modèle HTML de la section 4 qui inclut `<canvas>` et le `<script>` Chart.js. **Utilise uniquement** le modèle ci-dessous pour le champ `html`.
- **Interdiction stricte** : aucune balise `<canvas>`, aucun `<script>` (y compris chargement chart.js).
- Remplace les placeholders par des valeurs réelles : `[HH:MM:SS]`, `[Analysis Title]`, `[Key Insights]`, colonnes / lignes du tableau, puces des recommandations. Le tableau peut être **omis** si une simple liste à puces suffit (dans ce cas, retire le bloc « Data Table » ou mets un court paragraphe à la place).
- **Règle d’or (assouplie)** : synthèse courte ; chiffres clés et transparence minimale (source / requête).
- **Outils / SQL** : enchaînement court ; une requête bien ciblée quand elle suffit.
- Les champs structurés (`html`, `resultatSQL`, `formuleKPI`, `dataKPI`, `requeteSQL`) restent **obligatoires**.

### Modèle de page (mode rapide — sans graphique)
Reproduis la **structure, l’ordre des blocs et les styles inline** du modèle suivant.

```html
<div style="max-width:1000px; margin:20px auto; page-break-inside: avoid;">
  <!-- Real-time indicator -->
  <div style="background:transparent; padding:12px 16px; border-radius:8px; margin:16px 0; border-left:3px solid #4e79a7; display:flex; align-items:center; gap:8px; border-top:1px solid #444; border-right:1px solid #444; border-bottom:1px solid #444;">
    <span style="color:#4e79a7; font-size:16px;">🔍</span>
    <p style="color:#e0e0e0; margin:0; font-size:13px;">
      <strong style="color:#4e79a7;">Real-time Verification</strong> executed at [HH:MM:SS]
    </p>
  </div>

  <!-- Header & Insights -->
  <div style="background:transparent; padding:24px; border-radius:12px; border-left:4px solid #4e79a7; margin-bottom:20px; border:1px solid #444;">
    <h2 style="color:#4e79a7; margin:0 0 12px 0;">[Analysis Title]</h2>
    <p style="color:#e0e0e0; line-height:1.6; margin:0;">[Key Insights]</p>
  </div>

  <!-- Data Table -->
  <div style="background:transparent; padding:24px; border-radius:12px; overflow-x:auto; margin-bottom:20px; border:1px solid #444;">
    <table style="width:100%; border-collapse:collapse; background:transparent;">
      <thead>
        <tr style="background:transparent;">
          <th style="padding:12px; color:#e0e0e0; border-bottom:2px solid #4e79a7; text-align:left;">[Col]</th>
        </tr>
      </thead>
      <tbody>
        <tr style="border-bottom:1px solid #444;">
          <td style="padding:12px; color:#e0e0e0;">[Val]</td>
        </tr>
      </tbody>
    </table>
  </div>

  <!-- Recommendations -->
  <div style="background:transparent; padding:24px; border-radius:12px; border-left:4px solid #5cb85c; border:1px solid #444;">
    <h3 style="color:#5cb85c; margin:0 0 12px 0;">💡 Recommendations</h3>
    <ul style="color:#e0e0e0; line-height:1.8; margin:0; padding-left:20px;"><li>[Action]</li></ul>
  </div>
</div>
```

Un bloc **formules** (texte) peut compléter la page **après** le bloc recommandations si les consignes métier l’exigent, en reprenant le même style (transparent, bordure discrète).
